Présentation du sujet

L’objectif de ce TP est de faire un programme capable de bien jouer à Mastermind en Prolog.

On suposera que les prédicats gagne/2, n_bien_place/3 et n_mal_place/3 ont été implémentés lors de la séance précédente. Néanmoins, il est possible de réaliser le TP sans n_mal_place juste en omettant.

Pour cela on utilisera une variation de l’algorithme teste et génère et se baser sur les précédentes propositions faites.

Rappel du codage utilisé

Les codes seront représentés par des listes d’entiers. On numérotera les couleurs de 1 à M et la taille des codes sera notée N.

Rappel sur les couleurs bien placées et mal placées

Prenons le cas où N = 4 et M = 5. Supposons que le code soit [1, 2, 3, 4] et le décodeur propose [1, 3, 2, 3]. Le nombre de bien placées sera de 1 (le premier 1) et le nombre de mal placées sera de 2 (le premier 3 et le 2). Une couleur ne peut pas intervenir deux fois dans un calcul de couleurs bien placées ou mal placées. Une couleur bien placée n’intervient plus pour le calcul des couleurs mal placées. Le tableau suivant apporte quelques exemples supplémentaires.

Code initial Code proposé BP MP
[1, 2, 3, 4] [1, 2, 3, 4] 4 0
[1, 2, 3, 4] [1, 1, 1, 1] 1 0
[1, 2, 3, 4] [1, 1, 1, 2] 1 1
[1, 2, 3, 4] [1, 3, 2, 3] 1 2
[1, 2, 3, 4] [4, 3, 2, 1] 0 4

2 remarques importantes

  1. On peut remarquer que les 2 codes jouent un rôle symétrique dans le calcul de BP et de MP : les résultats restent les mêmes si l’on inverse le code initial et le code proposé.
  2. Il peut également être remarqué qu’une proposition est possible si elle est compatible avec l’ensemble des propositions précédentes, c’est à dire qu’elle a le même nombre de couleurs bien placées et le même nombre de couleurs mal placées que celles données par les propositions précédentes.

Algorithme generate and test appliqué au Mastermind

On va implémenter le joueur decodeur(+M, +N, +Hist, -Code) suivant l’approche générer et tester vue précédemment suivant le schéma dont le code est le suivant :

decodeur(M, N, Hist, Code) :-
    gen(M, N, Code),
    test(Code, Hist),
    !.

Question 1 : Quelle est la différence entre la version vue en cours et la version proposée ici ? Quelle est la conséquence sur le résultat calculé ?

Création du générateur

Ici, générer a pour objet de décrire l’ensemble des solutions potentielles sous la forme de l’ensemble des codes de taille NN comportant des couleurs prises dans [1..M][1..M], i.e. X=[1..M]NX=[1..M]N.

Question 2 : créer le prédicat liste_couleurs(+Min, +Max, -L) qui unifie L avec la liste des nombre de Min à Max (inclus).

Exemple :

?- liste_couleurs(2, 5, L).
L = [2, 3, 4, 5]`

Question 3 : Créer le générateur gen(+M, +N, -Code)qui . Utiliser pour cela member/2

Exemple :

?- gen(2, 3, L).
L = [1, 1, 1];
L = [1, 1, 2];
L = [1, 2, 1];
L = [1, 2, 2];
L = [2, 1, 1];
L = [2, 1, 2];
L = [2, 2, 1];
L = [2, 2, 2].

Création du test principal

Le prédicat test(+Code, +Historique) va écarter (filtrer) les éléments de XX qui sont incompatibles avec l’historique des propositions passées.

  • au niveau 0, on écarte les solutions déjà proposées, ce qui suffit à assurer la convergence
  • au niveau 1, on ne considère que les codes susceptibles de donner le même nombre d’éléments bien placés que les tentatives précédentes
  • au niveau 2, on cherche aussi à reproduire le nombre d’éléments mal placés

Exemple d’historique :

[prop([1, 3, 1, 2], 2, 2), 
 prop([1, 1, 2, 2], 2, 1), 
 prop([1, 1, 1, 1], 2, 0)]

On remarquera l’usage de la fonction prop qui permet de typer élégament le triplet (Code, Nombre_bien_placés, Nombre_mal_placés).

Question 4 : créer le prédicat test(+Code, +Historique) qui est vrai si et seulement si le code est compatible avec l’historique.

Exemple de test :

?- test([1, 3, 2, 1], 
     [prop([1, 3, 1, 2], 2, 2), 
      prop([1, 1, 2, 2], 2, 1), 
      prop([1, 1, 1, 1], 2, 0)]).
true.

Implémentation du jeu

Le jeu est lancé via une requête à l’arbitre : arbitre(+N, +M, +X, ?Hist). La variable X contient le code à trouver (par exemple N = 4, M = 3 et X = [3, 1, 1, 2]). L’historique Hist est initialisé à [].

Vous trouverez ci-après le code permettant de gérer le cas (d’arrêt) où le décodeur trouve le code :

arbitre(M, N, Code_a_trouver, Hist) :-
    decodeur(M, N, Hist, C ),
    gagne(Code_a_trouver, C),
    longueur(Hist, LH),
    write(C), nl,
    write("Gagné en "), write(LH), write(" coups !"), nl,
    !.

Question 5: Implémenter, sur la même base, le cas où le décodeur se trompe, en n’oubliant pas de mettre à jour l’historique.
